Python 函数 ​

在编程的世界里，函数（Function） 是构建复杂程序的基石。它不仅仅是一段带名字的代码块，更是我们实现代码复用、逻辑解耦的利器。

定义与文档字符串 ​

定义一个函数非常简单，使用 def 关键字即可。但一个好的函数，必须包含清晰的文档字符串（Docstring）。

  • def 关键字：告诉 Python 我要定义一个函数了。
  • 文档字符串：位于函数头下方的三引号（或双引号）内容。它不是普通的注释，而是用于描述函数“做什么”的说明书，这对于后期维护至关重要。它会被存储在函数的 __doc__ 属性中，可以通过 help() 函数调用，是自动化文档工具（如 Sphinx）的基础。
python
def greet_user():
    """显示简单的问候语"""
    # 这里的三引号内容就是文档字符串，可以通过 help(greet_user) 查看
    print("Hello!")

greet_user() # 调用函数

小贴士：养成写文档字符串的习惯，这会让你的代码看起来更专业，也更容易被他人理解。

形参与实参 ​

函数不是封闭的孤岛，它需要接收外部数据进行处理。这里涉及两个概念：

  • 形参 (Parameter)：函数定义时的变量，是接口声明。
  • 实参 (Argument)：调用函数时传递的具体信息（常量或变量）。

Python 提供了灵活的传参方式，让你不再为参数顺序头疼：

传参方式描述注意事项
位置实参实参与形参位置一一对应顺序必须严格一致，否则逻辑会出错。
关键字实参通过 形参名=实参值 传递无需关注顺序，但形参名必须拼写正确。
默认值定义时给形参赋默认值若调用时未传参，则使用默认值。

Python 的参数传递机制 Python 的参数传递既不是单纯的“值传递”也不是“引用传递”，而是**“对象引用传递”**。

  • 如果传递的是不可变对象（如数字、字符串、元组），函数内部修改形参不会影响外部实参（类似值传递）。
  • 如果传递的是可变对象（如列表、字典），函数内部修改形参会直接影响外部实参（类似引用传递）。

️关于默认值的避坑指南： 如果你给某个形参设置了默认值，那么该形参后面的所有形参都必须设置默认值。这是 Python 的语法规则，否则会报错。

可变默认参数 千万不要使用可变对象（如列表 [] 或字典 {}）作为默认参数！ 因为默认参数在函数定义时只计算一次。如果修改了它，后续调用都会受到影响。 错误写法：def add_item(item, lst=[]): ...正确写法：def add_item(item, lst=None): if lst is None: lst = []

传递任意数量的实参 ​

在实际开发中，我们往往无法预知用户会传递多少个参数。Python 提供了两种强大的机制来处理这种情况：

1. 接收任意数量的位置实参 (*args) ​

在形参前加星号 *，Python 会将所有接收到的实参打包成一个元组 (Tuple)。

python
def make_pizza(*toppings):
    """打印顾客点的所有配料"""
    print(toppings) # toppings 在这里是一个元组

make_pizza('pepperoni')
# 输出: ('pepperoni',)

2. 接收任意数量的关键字实参 (**kwargs) ​

在形参前加双星号 **，Python 会将接收到的键值对打包成一个字典 (Dictionary)。

python
def build_profile(first, last, **user_info):
    """创建一个字典，其中包含我们知道的有关用户的一切"""
    user_info['first_name'] = first
    user_info['last_name'] = last
    return user_info

3. 混合使用与位置原则 ​

如果同时使用普通参数和 *args，建议将 *args 放在最后。如果 *args 后面还有参数，那么后续参数必须使用关键字实参的方式进行传递。

参数定义的顺序 在定义复杂函数时，参数必须遵循以下顺序： def func(标准参数, *args, 默认参数, **kwargs):

函数返回值 ​

函数处理完数据后，通常需要将结果返回给调用者。

有返回值的情况：

python
def get_formatted_name(first_name, last_name):
    full_name = f"{first_name} {last_name}"
    return full_name.title() # return 语句结束函数执行并返回值

没有返回值的情况 (注意事项)： 如果函数体内没有 return 语句，函数执行完毕后默认返回 None。这类函数通常用于执行动作（如打印信息），而不是计算数据。

返回多个值 Python 允许函数返回多个值，本质上是返回了一个元组。 return a, b 等同于 return (a, b)。调用时可以使用 x, y = func() 进行解包。

Lambda 表达式 ​

在 Python 中，除了使用 def 定义命名函数外，还有一种特殊的函数叫做匿名函数，即 Lambda 表达式。

什么是 Lambda？ ​

Lambda 是一个没有名字、只能写一行代码的微型函数。它适合用于那些简单、一次性、不需要复用的逻辑场景。

语法格式：

python
lambda 参数: 表达式

Lambda 与普通函数的对比： ​

特性普通函数 (def)Lambda 表达式
名称有名字匿名（无名字）
代码体可以写多行，包含各种逻辑只能写一行表达式
返回值需要显式写 return自动返回表达式结果，无需 return
适用场景复杂业务逻辑、需要复用简单的数据处理、作为参数传递

代码示例：

python
# 普通函数写法
def add(x, y):
    return x + y

# Lambda 写法
add_lambda = lambda x, y: x + y

print(add(2, 3))       # 输出: 5
print(add_lambda(2, 3)) # 输出: 5

Lambda 的参数定义规则 ​

Lambda 表达式虽然简单，但其参数定义规则与普通函数（def）高度一致，只是省略了括号。

  1. 无参数： 如果逻辑不需要输入，可以定义空参数。

    python
    # 定义一个返回固定值的 lambda
get_pi = lambda: 3.14159
print(get_pi()) # 输出: 3.14159

  2. 多个位置参数： 参数之间用逗号 , 分隔。

    python
    # 计算长方形面积
area = lambda length, width: length * width
print(area(5, 3)) # 输出: 15

  3. 默认参数： 和普通函数一样，Lambda 也支持为参数指定默认值。

    python
    # 定义一个带有默认乘数的 lambda
multiplier = lambda x, factor=2: x * factor
print(multiplier(5))      # 使用默认值，输出: 10
print(multiplier(5, 3))   # 覆盖默认值，输出: 15

  4. 可变参数 (*args 和 **kwargs)： Lambda 也支持接收任意数量的参数，这在编写高阶函数时非常有用。

    python
    # 接收任意数量的位置参数并求和
sum_all = lambda *args: sum(args)
print(sum_all(1, 2, 3, 4)) # 输出: 10

# 接收任意数量的关键字参数
print_kwargs = lambda **kwargs: [f"{k}={v}" for k, v in kwargs.items()]
print(print_kwargs(a=1, b=2)) # 输出: ['a=1', 'b=2']

️ 注意：虽然 Lambda 写起来很酷，但不要滥用。如果逻辑稍微复杂一点，请务必使用 def 定义普通函数，以保证代码的可读性。

逻辑解耦 ​

掌握了函数的语法只是第一步，如何用好它们，写出高质量的代码，则需要理解逻辑解耦的思想。

什么是解耦？ ​

简单来说，解耦就是降低代码不同部分之间的依赖关系。一个高度解耦的系统，其各个模块就像独立的积木块，可以轻松地被替换、修改或复用，而不会轻易“牵一发而动全身”。

  • 紧耦合：就像一列焊死的火车，一个车厢出问题，整列车都动不了。
  • 松耦合：就像由独立服务组成的生态系统，通过定义好的接口通信。

如何实现逻辑解耦？ ​

关键在于依赖倒置原则：高层模块不应该依赖于低层模块，二者都应该依赖于抽象。

反面教材：紧耦合的代码

python
class ReportGenerator:
    def generate(self, data):
        # 问题：直接依赖了具体的 ExcelExporter
        # 如果以后想换成 PDF 导出，必须修改这里的代码，违反了“开闭原则”
        exporter = ExcelExporter()
        exporter.export(data)

正面教材：松耦合的代码（依赖注入）

python
class ReportGenerator:
    def __init__(self, exporter): # 注入抽象接口
        self._exporter = exporter

    def generate(self, data):
        # 只依赖 exporter 接口，不关心具体是 Excel 还是 PDF
        self._exporter.export(data)

这种将依赖从外部注入的设计模式，被称为依赖注入（Dependency Injection），是实现逻辑解耦最常用和有效的方法之一。

模块 (Modules) ​

当函数越来越多时，将所有代码堆在一个文件里会变得难以维护。这时我们需要引入模块的概念。

导入模块的四种姿势：

  1. 导入整个模块：import module_name
    • 调用时需加前缀：module_name.func()
  2. 导入特定函数：from module_name import function_name
    • 调用时直接：function_name()
  3. 使用别名：import module_name as md
    • 简化长模块名或避免冲突。
  4. 导入所有：from module_name import * (慎用)
    • 风险：会将模块中所有公开名称导入当前命名空间，极易导致命名覆盖（例如不小心覆盖了内置函数 open 或 list），且难以追踪变量来源。
